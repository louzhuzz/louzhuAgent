# 学习路线指导

这份文件是当前项目里**优先级最高的指导文件**。

用途不是替代所有知识点文档，而是帮你在继续开发和学习时快速回答这几个问题：

1. 现在到底处在什么阶段
2. 接下来先做什么、后做什么
3. 哪些内容是“理解底层”的示例
4. 哪些内容是“主项目真正要保留”的能力
5. 后面要不要接 LangChain 1.x、LangGraph、ChromaDB

如果你后面看路线犹豫了，先看这份文件，再回去看详细总纲。

同时再配合看：

- [知识点/01-项目包装目标.md](/mnt/d/AIcodes/Agent/知识点/01-项目包装目标.md)

因为从现在开始，后续迭代不只是为了“功能能跑”，还要逐步对齐“简历可写、面试可讲、指标可量化”的项目目标。

## 1. 当前项目的总目标

当前这个项目的目标不是做一个只会聊天的 Demo，而是逐步做出一个：

**个人学习助理 Agent**

它最终应该具备这些能力：

- 多轮记忆
- 结构化学习计划
- 任务拆解
- 知识点问答
- 本地工具调用
- Agent 决策
- 可追溯回答
- 可观测、可评测

## 2. 当前阶段判断

你现在已经完成了前面很重要的一段：

- 最小对话 Agent
- 多轮记忆
- 结构化输出
- Prompt 模板化
- LangChain 基础
- RAG 基础
- Tool Calling
- Tool Calling Agent
- ReAct Agent
- 主项目开始项目化重构

所以你现在已经**不是入门期**，而是进入了：

**项目化实战期**

前面学的是“技术点”，  
现在做的是“把技术点收敛成主项目能力”。

## 3. 从现在开始的最高原则

从现在开始，所有开发和学习都优先遵守这几个原则：

### 原则 1：主项目和旧示例分开

- `src/`：主项目核心代码
- `examples/`：旧示例和教学脚本

以后再做新能力时，先判断：

- 这是主项目能力？
- 还是教学示例？

### 原则 2：前面的示例用于理解底层，后面的项目用于收敛工程能力

也就是说：

- 示例不等于最终项目实现
- 主项目不需要把所有示例原样搬进去

### 原则 3：LangChain 统一按 1.x 理解

后面只要涉及框架主线，统一按：

- `LangChain 1.x`
- `langchain-openai 1.x`
- `LangGraph`

去理解。

### 原则 4：RAG 要考虑成本

当前项目已经明确有真实约束：

- embedding token 成本高

所以主项目默认策略是：

- 不做全库无脑 embedding
- 优先少量候选文件筛选
- 再做小范围 embedding 检索
- 后面再逐步做缓存和持久化

### 原则 5：后续能力迭代要对齐项目包装目标

也就是说，后面每做一步，不只是看：

- 功能有没有实现

还要看：

- 能不能写进简历
- 能不能讲进面试
- 能不能形成指标化结果

对应目标文件：

- [知识点/01-项目包装目标.md](/mnt/d/AIcodes/Agent/知识点/01-项目包装目标.md)

## 7. ChromaDB 的目标位置

这一点你特别提了，所以单独写清楚。

### 现在为什么还没接 ChromaDB

因为当前主项目先走的是：

- 少量候选文件
- 小范围 embedding 检索

这有利于：

- 控制 token
- 控制复杂度
- 先把主项目功能做成型

### 后面为什么一定要学 ChromaDB

因为当前项目后续如果继续升级，单靠临时内存向量库不够：

- 不能持久化
- 每次都可能重新建库
- 不利于缓存和长期复用

所以后面你要有一个明确目标：

**把当前知识点问答系统，升级成基于 ChromaDB 的持久化向量检索版本。**

### ChromaDB 在路线里的定位

ChromaDB 不属于最前面的基础课，它更适合放在：

- 主项目第一版完成之后
- 但在评测体系和长期记忆之前

在当前正式主线里，你可以把它理解成：

- 完成 `Agent v1` 之后的下一课
- 也就是总纲里的正式课程节点
- **第 27 步：Embedding 缓存与 ChromaDB 持久化知识库**

也就是一个比较合理的位置是：

1. 先完成主项目版学习计划 / 问答 / 任务拆解
2. 再把知识点问答系统升级为：
   - embedding 缓存
   - ChromaDB 持久化向量库
3. 然后再进入：
   - 日志
   - LangSmith
   - 评测

### 你后面关于 ChromaDB 的具体目标

建议你后面把 ChromaDB 当成一个明确阶段目标：

#### 目标 1：替换当前临时内存向量库

把：

- `InMemoryVectorStore`

升级成：

- `Chroma` / `ChromaDB`

#### 目标 2：实现 embedding 本地持久化

做到：

- 同样的知识点文件不重复 embedding
- 文件没变就直接复用

#### 目标 3：实现增量更新

做到：

- 新增知识点时只补新增文件
- 修改文件时只重建对应文档片段

#### 目标 4：为主项目问答提供稳定知识库层

这样后面你的 `/qa` 才真正具备：

- 持久化
- 可扩展
- 可长期维护

### 一句话结论

**ChromaDB 不是当前第一优先级，但它是主项目知识问答系统后续升级的明确目标。**

## 8. 现在最应该避免的事情

### 1. 不要同时开太多技术支线

比如：

- 一边做多 Agent
- 一边做 LangGraph
- 一边改主项目目录
- 一边重做 RAG

这样容易失控。

### 2. 不要把主项目重新写回示例风格

主项目现在已经开始有：

- 请求对象
- 服务层
- 项目化能力模块

后面不要再退回到：

- 所有逻辑堆在 `agent.py`

### 3. 不要忽略成本问题

embedding、上下文长度、重复调用，这些在主项目里都是真问题。

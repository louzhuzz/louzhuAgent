# 第一步：选什么项目最适合入门 Agent

## 结论

最适合你的第一个项目是：

**个人学习助理 Agent**

## 这个项目具体做什么

你输入一个学习问题，比如：

- `帮我解释一下 RAG 是什么`
- `根据我的笔记，总结 LangChain 的核心模块`
- `帮我制定 3 天的 Agent 学习计划`

Agent 会做这些事：

1. 理解你的目标
2. 判断需不需要查本地资料
3. 判断需不需要调用工具
4. 组织答案
5. 输出结果，并尽量给出依据

## 为什么不先做特别复杂的项目

很多人一上来就想做：

- 自动写代码的超级 Agent
- 多智能体协同平台
- 全自动办公助手

这类项目的问题不是“不高级”，而是：

- 依赖太多
- 调试太难
- 问题定位困难
- 初学者很容易只会“跑 demo”，却不理解核心机制

你现在更需要的是：

**一个足够简单，但能覆盖 Agent 核心能力的项目。**

## 这个项目能覆盖哪些核心知识点

它几乎可以把 Agent 入门常见模块都串起来：

- LLM 调用
- Prompt 设计
- 输出结构化
- Tool Calling
- RAG
- Memory
- Workflow
- 多 Agent
- 评估与调试

## 为什么它比普通聊天机器人更适合学习

普通聊天机器人通常只练到：

- 发送消息
- 返回回答

但 Agent 的重点不只是“能聊天”，而是：

- 会判断
- 会选择动作
- 会调用外部能力
- 会基于上下文持续完成目标

“个人学习助理 Agent”刚好天然适合这些能力。

## 第一阶段先做到什么程度

第一阶段不要贪大，先做到：

- 用户输入一个问题
- Agent 用固定 Prompt 理解问题
- 调用模型输出答案
- 把交互流程跑通

这就已经是一个最小可运行版本了。

## 当前阶段你需要记住的关键概念

### 1. Agent 不等于大模型

大模型负责“生成”。

Agent 负责“决策和执行流程”。

可以先记成这个公式：

`Agent = LLM + Prompt + Memory + Tools + Workflow`

### 2. 项目要能逐步生长

一个好的学习项目，不是功能一开始就很多，而是：

- 今天能做最小版本
- 明天能继续加能力
- 每一步都能验证

### 3. 先做单 Agent，再谈多 Agent

很多教程喜欢一上来讲多 Agent。

但如果单 Agent 的输入、输出、工具调用、状态管理都没搞清楚，多 Agent 只会让复杂度翻倍。

## 我们接下来怎么推进

下一步我建议直接进入：

**第二步：搭最小可运行版项目骨架**

会做这些事：

1. 选技术栈
2. 建目录结构
3. 写最小主程序
4. 接入一个模型
5. 跑通第一次问答

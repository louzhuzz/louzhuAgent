# Agent 核心概念速记

## 1. 什么是 Agent

先用最实用的理解：

**Agent 是一个能基于目标进行判断，并在必要时调用外部能力完成任务的系统。**

它不是单纯“回答一句话”，而是：

- 接收目标
- 分析问题
- 决定步骤
- 调用工具
- 整理结果

## 2. Agent 和普通聊天机器人的区别

普通聊天机器人更像：

- 你问一句
- 它答一句

Agent 更像：

- 你给一个任务
- 它决定要不要拆解
- 它决定要不要查资料
- 它决定要不要调用工具
- 它再给你结果

## 3. Agent 的典型组成

可以先记这一套：

### LLM

负责理解和生成文本。

### Prompt

负责告诉模型：

- 它是谁
- 它要干什么
- 输出要遵守什么格式

### Tools

让 Agent 不只会“说”，还会“做”。

例如：

- 搜索
- 读文件
- 调接口
- 查数据库

### Memory

让 Agent 保留上下文或长期记录。

### Workflow

规定执行顺序。

比如：

1. 先分析问题
2. 再决定是否检索知识库
3. 最后生成答案

## 4. 为什么 RAG 很重要

RAG 可以简单理解为：

**先检索资料，再让模型基于资料回答。**

它的价值是：

- 降低幻觉
- 让回答基于你的私有知识
- 让答案更可追溯

## 5. 初学者最容易混淆的点

### Agent 不是框架名

LangChain、LangGraph、AutoGen 这些是工具或框架。

Agent 是一种系统形态，不依赖某一个特定框架才成立。

### Tool Calling 不等于真调用成功

模型只能“建议调用哪个工具”。

真正调用工具、处理参数、接收结果，还是要靠你的程序逻辑。

### 多 Agent 不是越早做越好

如果单 Agent 还不稳定：

- 多 Agent 只会更难调试
- 更难知道错误出在哪一层

## 6. 这一阶段你应该建立的思维

不要追求“大而全”，先追求：

- 能跑
- 可解释
- 可验证
- 可迭代

如果每一步你都能回答这三个问题，进步会很快：

1. 这一步输入是什么？
2. 这一步输出是什么？
3. 这一步为什么需要存在？

# 第十六步：文档加载

## 1. 本节目标

这一节正式进入 RAG。

但我们不先讲向量库，也不先讲检索。

先做最基础的一步：

**把本地 Markdown 文档加载成统一的文档对象。**

本节新增的示例文件是：

- [src/document_loading_intro.py](/mnt/d/AIcodes/Agent/src/document_loading_intro.py)

## 2. 是什么

文档加载，就是把磁盘上的原始文件读进程序，并转成后续 RAG 流程能处理的统一结构。

在 LangChain 语境里，一个文档对象最常见的两个核心部分是：

- `page_content`
- `metadata`

你可以先把它理解成：

- `page_content`：文档正文
- `metadata`：文档附加信息

## 3. 为什么

RAG 的第一步不是“检索”，而是“先有文档对象”。

如果文档还只是磁盘上的原始文件，你后面这些步骤都做不了：

- 切分
- 向量化
- 建索引
- 检索
- 引用来源

所以文档加载是整个 RAG 管道的入口。

## 4. 怎么做

### 第一步：确定输入目录

我们当前项目最合适的输入源就是：

- [知识点](/mnt/d/AIcodes/Agent/知识点)

因为这里已经有一批现成的 Markdown 文档。

### 第二步：逐个读取 `.md` 文件

这一步做的事情很朴素：

- 找到目录中的每个 `.md`
- 读取文本内容

### 第三步：把文本包装成 `Document`

本节我们使用：

```python
from langchain_core.documents import Document
```

把每个文件变成统一对象。

### 第四步：补上 metadata

metadata 很重要，因为后面你要做这些事时会依赖它：

- 显示来源文件
- 做引用
- 排查检索结果

所以本节先放了这些字段：

- `source`
- `file_name`
- `suffix`

## 5. 关键代码

### 代码 1：文档对象构造

文件：[src/document_loading_intro.py](/mnt/d/AIcodes/Agent/src/document_loading_intro.py)

```python
Document(
    page_content=content,
    metadata={
        "source": str(path.relative_to(PROJECT_ROOT)),
        "file_name": path.name,
        "suffix": path.suffix,
    },
)
```

这段代码是本节最核心的部分。

它说明一件事：

**RAG 里的文档不是裸字符串，而是“正文 + 元信息”的对象。**

### 代码 2：批量加载 Markdown

文件：[src/document_loading_intro.py](/mnt/d/AIcodes/Agent/src/document_loading_intro.py)

```python
for path in sorted(directory.glob("*.md")):
    content = path.read_text(encoding="utf-8").strip()
```

这段代码做的事是：

- 扫描目录
- 逐文件读取
- 按顺序稳定处理

### 代码 3：预览文档内容

文件：[src/document_loading_intro.py](/mnt/d/AIcodes/Agent/src/document_loading_intro.py)

```python
preview = document.page_content[:120].replace("\n", " ")
print(f"metadata: {document.metadata}")
print(f"preview: {preview}...")
```

这段代码的价值是：

- 帮你验证文档到底有没有加载成功
- 帮你看到 `page_content` 和 `metadata` 的真实样子

## 6. 常见错误

### 错误 1：把文档只当成纯字符串

后果：

- 后面做来源引用时会很难受
- 无法方便地附带文件信息

### 错误 2：不存 metadata

后果：

- 检索到了内容，也不知道来自哪个文件

### 错误 3：加载阶段就做太多复杂处理

这一节的重点只是：

- 先读进来
- 先统一结构

不要一上来把切分、清洗、向量化全塞进去。

## 7. 常见面试问题

### 问题 1：RAG 为什么要先做文档加载？

回答要点：

- 检索和向量化都依赖统一文档对象
- 文档加载是整个知识管道的入口
- 没有文档对象，后面流程无法稳定展开

深入追问：

- 为什么 RAG 不应该直接对原始文件路径做检索？

### 问题 2：`page_content` 和 `metadata` 分别负责什么？

回答要点：

- `page_content` 存正文内容
- `metadata` 存来源、文件名、标签等附加信息
- 两者一起决定文档对象的可用性

深入追问：

- 哪些 metadata 对后续引用和评测最有价值？

### 问题 3：为什么这一节先手写加载逻辑，而不是直接上社区 Loader？

回答要点：

- 先理解文档对象本质
- 先看清加载阶段到底做了什么
- 后面再上 Loader 框架更容易理解抽象

深入追问：

- 什么时候适合迁移到 `langchain_community.document_loaders`？

### 问题 4：文档加载和文本切分的边界在哪里？

回答要点：

- 文档加载负责把原始文件变成统一对象
- 文本切分负责把长文档拆成更适合检索的小块
- 这是两个不同阶段，不应该混在一起

深入追问：

- 为什么切分放在加载后而不是加载前？

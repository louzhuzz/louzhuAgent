# 第二十七步：第一个工具函数

## 1. 本节目标

这一节的目标不是马上让模型“自动调用工具”，而是先把最基础的问题讲透：

- 工具到底是什么
- 工具由谁执行
- 工具和普通聊天有什么本质区别

本节会先实现两个最小本地工具：

- `list_notes()`
- `read_note(file_name)`

对应代码文件：

- [src/tools.py](/mnt/d/AIcodes/Agent/src/tools.py)
- [src/agent.py](/mnt/d/AIcodes/Agent/src/agent.py)
- [src/main.py](/mnt/d/AIcodes/Agent/src/main.py)

## 2. 是什么

在 Agent 语境里，工具就是：

**模型之外、由程序真实执行的能力。**

比如：

- 读文件
- 查天气
- 调数据库
- 发 HTTP 请求

这些能力不是模型“想一想”就能完成的，必须由你的 Python 代码真正执行。

所以这一节做的两个工具，本质上是：

- 列出本地知识点文件
- 读取某个知识点文件内容

## 3. 为什么

你前面已经有：

- 聊天
- 结构化输出
- RAG

但这些能力里，模型主要还是在“生成内容”。

现在要迈向 Agent，就必须开始理解一个非常关键的边界：

**模型负责决定，程序负责执行。**

这一节虽然还没有进入“模型自动选择工具”，但它先让你把这层边界学清楚。

## 4. 怎么做

### 第一步：新建 `src/tools.py`

我们把工具单独放在一个文件里，而不是直接塞进 `main.py`。

这样做的好处是：

- 工具职责更集中
- 后面做 Tool Calling 时更容易扩展
- 不会把主程序写成一大坨

### 第二步：实现 `list_notes()`

这个函数负责：

- 扫描 `知识点/` 目录
- 找出所有 `.md` 文件
- 返回文件名列表

### 第三步：实现 `read_note(file_name)`

这个函数负责：

- 读取指定 Markdown 文件
- 并做最基本的安全限制

这里的安全限制包括：

- 不能为空
- 不允许带路径分隔符
- 只允许读 `.md`
- 文件必须存在

### 第四步：在 `LearningAgent` 中增加工具方法

这一层的目的不是让 Agent 自动调用工具，而是先给 Agent 一个清晰的调用入口：

- `list_notes_tool()`
- `read_note_tool(file_name)`

### 第五步：在命令行入口增加 `/notes` 和 `/read`

这样你现在就能直接在命令行里体验工具：

```text
/notes
/read 13-输出解析器.md
```

## 5. 关键代码

### 代码 1：列出知识点文件

文件：[src/tools.py](/mnt/d/AIcodes/Agent/src/tools.py)

```python
def list_notes() -> list[str]:
    """列出知识点目录下所有 Markdown 笔记文件名。"""
    return sorted(path.name for path in KNOWLEDGE_DIR.glob("*.md"))
```

详细理解：

- `KNOWLEDGE_DIR.glob("*.md")`
  作用：扫描知识点目录下所有 Markdown 文件
- `path.name`
  作用：只拿文件名，不拿完整路径
- `sorted(...)`
  作用：让输出结果稳定，方便阅读和测试

为什么这里返回的是“文件名列表”而不是“完整路径列表”？

因为给用户看的时候，文件名更直观；同时这也减少了路径暴露。

### 代码 2：读取单个知识点文件

文件：[src/tools.py](/mnt/d/AIcodes/Agent/src/tools.py)

```python
def read_note(file_name: str) -> str:
    """读取指定知识点笔记内容，并限制只能访问知识点目录下的 Markdown 文件。"""
    cleaned_name = file_name.strip()
    if not cleaned_name:
        raise ValueError("文件名不能为空。")

    if "/" in cleaned_name or "\\" in cleaned_name:
        raise ValueError("只允许传入文件名，不允许传入路径。")

    if not cleaned_name.endswith(".md"):
        raise ValueError("当前工具只允许读取 .md 文件。")

    note_path = KNOWLEDGE_DIR / cleaned_name
    if not note_path.exists():
        raise FileNotFoundError(f"未找到知识点文件：{cleaned_name}")

    return note_path.read_text(encoding="utf-8").strip()
```

这段代码要重点理解两件事：

第一件事：它不只是“读文件”，还在做输入约束。

为什么？

因为工具不是给人手工点点就结束的，后面它会变成“模型也可能发起调用”的能力。

一旦工具没有边界，后面就很容易出现：

- 读错文件
- 读到不该读的路径
- 被路径穿越攻击

第二件事：为什么限制“不允许传路径，只允许传文件名”？

因为如果允许像下面这样传值：

```text
../../something.txt
```

就有可能绕出 `知识点/` 目录。

所以当前这个版本，明确要求：

- 只能传文件名
- 不能传路径

### 代码 3：Agent 层暴露工具方法

文件：[src/agent.py](/mnt/d/AIcodes/Agent/src/agent.py)

```python
def list_notes_tool(self) -> list[str]:
    """执行列出知识点文件的本地工具。"""
    return list_notes()

def read_note_tool(self, file_name: str) -> str:
    """执行读取指定知识点文件内容的本地工具。"""
    return read_note(file_name)
```

为什么不直接在 `main.py` 里调 `tools.py`？

因为当前项目里，`LearningAgent` 是你的核心业务对象。

把工具入口先收敛到 Agent 类上，后面你做：

- Tool Calling
- Agent 自动决策

时会更自然。

### 代码 4：命令行触发工具

文件：[src/main.py](/mnt/d/AIcodes/Agent/src/main.py)

```python
if user_input == "/notes":
    notes = agent.list_notes_tool()
    print("\n知识点文件:")
    for note in notes:
        print(f"- {note}")
    continue
```

这段代码说明：

- 当前“是否调用工具”的决定，是命令行程序做的
- 还不是模型自动做的

再看读取文件：

```python
if user_input.startswith("/read "):
    file_name = user_input[6:].strip()
    ...
    content = agent.read_note_tool(file_name)
```

数据流是：

1. 用户输入 `/read 13-输出解析器.md`
2. `main.py` 识别出这是一个工具命令
3. 调 `agent.read_note_tool(file_name)`
4. Agent 再调用 `tools.read_note(file_name)`
5. Python 代码真实读取文件并返回内容

## 6. 常见错误

### 错误 1：以为“工具”就是让模型回答得更详细

不是。

工具不是“让模型多说两句”，而是：

- 让程序执行模型自己做不到的事

### 错误 2：把工具逻辑直接写死在 Prompt 里

这不是真工具。

Prompt 只能告诉模型“你可以做什么”，不能让它真的访问本地文件系统。

### 错误 3：读文件工具没有做路径限制

这会产生安全问题。

当前这节虽然是入门版，但已经先做了这层最基本的约束。

### 错误 4：以为“写了工具函数”就等于 Tool Calling 完成了

还没有。

这一节只是：

- 先把工具实现出来
- 先让程序能调用

下一阶段才会进入：

- 工具 Schema
- 模型决定是否调用工具

## 7. 常见面试问题

### 问题 1：Agent 里的工具和普通 Python 函数有什么区别？

回答要点：

- 从实现上看，工具通常就是普通函数
- 从系统设计上看，它是“暴露给 Agent 的外部能力”
- 所以它要考虑输入、输出、权限、错误处理

深入追问：

- 为什么不是所有函数都适合直接暴露成 Agent 工具？

### 问题 2：为什么读文件工具一定要做路径限制？

回答要点：

- 因为模型输入不可信
- 工具一旦没有边界，就可能访问到不该访问的资源
- 工具风险常常比“回答错了”更高

### 问题 3：为什么这一节先做命令式工具，而不是立刻做模型自动调用？

回答要点：

- 因为要先把“工具是什么、谁执行、数据怎么流动”学清楚
- 如果一上来就做自动调用，容易只会用框架，不理解底层边界

# 第二十二步：RAG v2 引用与可追溯回答

## 1. 本节目标

这一节是在 RAG v1 的基础上做一个关键升级：

- 让回答带来源
- 让回答可追溯

本节新增的示例文件是：

- [examples/rag_v2_citations_intro.py](/mnt/d/AIcodes/Agent/examples/rag_v2_citations_intro.py)

## 2. 是什么

引用与可追溯回答，指的是：

模型回答时不只是“给答案”，还要尽量说明：

- 这个结论来自哪个参考片段

在这一节里，我们采用的最小方式是：

- 用 `[片段 1]`、`[片段 2]` 这样的编号做引用

## 3. 为什么

RAG v1 已经能做到：

- 基于检索结果回答

但如果回答没有来源标注，用户仍然很难判断：

- 哪句话是基于哪段资料
- 依据到底充分不充分

所以 RAG v2 的关键价值是：

**把“有资料”升级成“有证据链”。**

## 4. 怎么做

### 第一步：在上下文里明确编号和来源

这一节我们增强了上下文构造逻辑。

现在每个片段不仅有：

- 片段编号
- 来源文件

还有：

- `chunk_index`

### 第二步：在 Prompt 里明确要求引用

这一节最关键的升级点之一是：

Prompt 不再只说“基于资料回答”，而是明确要求：

- 核心结论尽量使用 `[片段X]` 标注

### 第三步：把来源清单单独打印出来

这一步不是为了好看，而是为了让你更容易验证：

- 回答里的 `[片段 1]`
- 到底对应的是哪一段真实资料

## 5. 关键代码

### 代码 1：增强后的上下文

文件：[examples/rag_v2_citations_intro.py](/mnt/d/AIcodes/Agent/examples/rag_v2_citations_intro.py)

```python
parts.append(
    f"[片段 {index}] 来源: {source} | chunk_index: {chunk_index}\n{content}"
)
```

这一步的意义是：

- 不只给模型正文
- 还把引用所需的编号和来源一起给进去

### 代码 2：Prompt 中加入引用要求

文件：[examples/rag_v2_citations_intro.py](/mnt/d/AIcodes/Agent/examples/rag_v2_citations_intro.py)

```python
4. 每一个核心结论后，尽量用 [片段X] 的形式标注依据
5. 如果某个结论没有足够依据，不要强行下结论
```

这两条要求决定了这一节和 RAG v1 的本质差别。

文件：[examples/rag_v2_citations_intro.py](/mnt/d/AIcodes/Agent/examples/rag_v2_citations_intro.py)

```python
    ...
```

这一步的作用是：

- 让你人工核对引用是否合理
- 为后面做引用评测打基础

## 6. 常见错误

### 错误 1：让模型“引用”，但上下文里没有清晰编号

后果：

- 模型可能随便编 `[片段 3]`
- 引用无法对应真实资料

### 错误 2：回答里写了引用，但不提供来源清单

后果：

- 用户还是难以追溯

### 错误 3：把“引用”误解成已经完全可靠

不是。

模型即使带引用，也仍然可能：

- 引错片段
- 过度概括
- 做出不充分推断

## 7. 常见面试问题

### 问题 1：为什么 RAG 需要可追溯回答？

回答要点：

- 让答案更可解释
- 让用户能核对依据
- 有助于降低“看起来对但其实没依据”的风险

深入追问：

- 可追溯回答和“减少幻觉”之间是什么关系？

### 问题 2：为什么 RAG v2 比 v1 更有工程价值？

回答要点：

- v1 只是基于资料回答
- v2 进一步提供证据链
- 更适合做真实产品和调试

深入追问：

- 为什么很多生产系统要求必须展示来源？

### 问题 3：带引用就一定可靠吗？

回答要点：

- 不一定
- 模型可能错误引用或过度推断
- 引用只是提高可验证性，不是绝对正确性保证

深入追问：

- 怎么评估“引用是否真的支持结论”？

### 问题 4：为什么来源信息要放进 metadata 和上下文里？

回答要点：

- metadata 方便程序追溯
- 上下文中的编号方便模型引用
- 两者配合才能形成完整证据链

深入追问：

- 如果要做更精细的引用，metadata 里还应存什么？
